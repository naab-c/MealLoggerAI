1. Information We Collect

Meal Logger AI allows users to enter information about meals they have eaten. - Meal Data: Text descriptions of foods (e.g., "2 boiled eggs, 1 cup yogurt"). - User Settings: Email and username stored locally in Chrome storage to map the user to their dedicated tab in the shared Google Sheet.

2. How We Use Information

Your meal data is used only for:

  • Providing nutrition analysis (calories, protein, carbs, fats) via the OpenAI API.
  • Saving entries into a shared Google Sheet where each approved user has their own tab.
No other use of your information is made.

3. Data Sharing

We do not sell or share your data with third parties. Data is only transmitted securely to:

  • Google Apps Script backend: used as a proxy to relay requests (due to CORS restrictions).
  • OpenAI API: to generate nutrient breakdowns.
  • Google Sheets: to store logged meals in a shared spreadsheet managed by the sheet owner.

4. Security

- All communication is encrypted with HTTPS.
- Only the sheet owner and approved users have access to the shared sheet.
- Data is never used beyond the stated purpose.

5. User Control and Data Deletion

- Users can stop using the extension at any time.
- Users can request removal from the shared Google Sheet by contacting the sheet owner.
- No other personal or sensitive data is collected or stored.

6. Remote Code

The extension does not download or execute remote code. It only makes secure HTTPS requests to the proxy backend (Google Apps Script) to relay meal data to the OpenAI API.
